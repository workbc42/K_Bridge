# 추천/재주문 MVP UI·플로우

Manychat 운영 + Beeper 베타 병행을 전제로 한 **추천/재주문 기능 MVP UI·플로우**입니다.

## 목표
- 주문 입력 시간을 단축
- 자주 주문한 메뉴를 빠르게 재주문
- 특이사항을 자동 제안

## 고객 흐름 (메시지 기반)

### A) 재주문 플로우
1. 고객 DM 수신
2. 시스템이 최근 주문 요약 자동 응답
3. 버튼: `그대로 재주문` / `수정해서 주문`
4. 그대로 재주문 → 주문 확정 정보만 요청 (주소/변경사항)

### B) 추천 플로우
1. 고객 DM 수신
2. 시스템이 “자주 주문한 메뉴 Top 3” 제안
3. 버튼: 메뉴 선택 → 수량/옵션 입력 → 주문 확정

### C) 특이사항 제안
1. 주문 확정 직전
2. “평소 사용하던 요청사항” 자동 제안
3. 버튼: `그대로 사용` / `수정`

## 내부 운영 플로우
- 주문 접수 화면에서 “재주문/추천” 라벨 표시
- request_note 자동 채움
- 고객 히스토리 링크 (최근 3건 요약)

## 최소 UI 구성 (내부 대시보드)

### 주문 카드 (칸반)
- 배지: `재주문` / `추천` 표시
- 히스토리: 최근 주문 요약 1줄

### 상세 모달
- 최근 주문 3건 요약
- 자주 주문한 메뉴 Top 3
- 특이사항 추천 목록
- 클릭 시 주문 상세에 자동 삽입

## MVP 적용 범위
- 추천/재주문은 **DM 텍스트 응답 + 내부 대시보드 보조 정보**까지만
- 결제/완전 자동 주문은 제외

## 측정 지표
- 평균 주문 처리 시간 감소
- 재주문 버튼 클릭률
- 특이사항 재사용률

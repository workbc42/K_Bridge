# 내부 대시보드 기획/설계 정리

아래 내용은 현재 프로젝트 문서와 프리뷰 코드를 기반으로 한 **내부 대시보드 기획안 초안**입니다. 추후 프리뷰 관련 내용을 계속 확장할 수 있도록 구조화했습니다.

## 1) 현재 상황 분석

### 프로젝트 특성
- 목적: 외국인을 위한 한국 배달앱 주문 대행 서비스
- 운영 방식: 메신저(Instagram/Facebook) 주문 접수 → 운영자 수동 대행
- 핵심 프로세스: 주문 접수 → 번역/확인 → 배달앱 주문 → 배달 추적 → 완료

### 프리뷰 코드 상태
- Kanban 기반 주문 관리 (접수/처리중/완료)
- 카드 UI + 상세 모달
- 고객 인사이트(최근 주문/자주 주문/특이사항) 표시

**강점**
- 상태 관리 직관성
- 정보 가시성
- 추천/재주문 초안 반영 기능

**한계**
- 실시간 연동 없음
- Manychat 통합 없음
- 결제/정산 기능 없음
- 다국어 메시지 처리 미구현

---

## 2) 내부 대시보드 핵심 요구사항

### 주문 처리 플로우
1. 메신저 접수(Manychat)
2. 주문 확인/번역
3. 배달앱 주문 대행
4. 고객 알림
5. 배달 추적
6. 완료 확인

### 필수 기능
- 주문 접수 및 상태 관리
- 고객 정보 및 주문 히스토리
- 실시간 알림 (신규 주문/이슈)
- 배달 추적 (수동/연동)
- 결제 확인 및 정산
- 다국어 지원
- 통계/리포트

---

## 3) 추천 기술 스택(정리)

### Frontend
- Next.js (App Router)
- Tailwind CSS
- 상태 관리: Zustand
- 데이터 패칭: TanStack Query
- 다국어: i18next
- 실시간: Socket.io

### Backend
- Node.js + Express
- PostgreSQL
- Redis (세션/캐시)
- 인증: JWT + OAuth2
- API: REST + WebSocket

### 외부 연동
- Manychat API
- 카카오 알림톡
- Google Translate API
- 배달앱 수동 대행 → 장기적으로 API 제휴

### DevOps
- GitHub Actions
- AWS Lightsail → EC2 확장
- Sentry, GA

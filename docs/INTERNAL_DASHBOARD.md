# 내부 주문 접수 대시보드 프리뷰

이 문서는 Manychat + Notion + Slack 단계의 MVP 운영을 위한 **내부 주문 접수 화면(칸반 대시보드)** 구현 내용과 사용법을 정리합니다.

## 목표
- 고객은 Messenger/Instagram( Manychat )으로 주문 스크린샷과 정보를 전달
- 내부 운영팀이 주문을 접수/처리/완료 상태로 관리
- 현재 버전은 UI/UX 검증용 **목데이터 기반**

## 구현 위치
- UI: `frontend/app/dashboard/orders/page.js`
- 레이아웃: `frontend/app/dashboard/layout.js`
- 스타일: `frontend/app/globals.css`
- 공통 컴포넌트: `frontend/components/layout/*`
- i18n 리소스: `frontend/messages/*.json`
- i18n 유틸: `frontend/lib/i18n.js`, `frontend/lib/i18n-client.js`
- 목데이터: `frontend/lib/mockOrders.js`

## 주요 기능
1. 칸반형 주문 리스트
2. 주문 카드에 스크린샷 썸네일(더미) 표시
3. 카드 클릭 또는 버튼 클릭으로 상세 모달 열기
4. 모달 외부 클릭 또는 `Esc` 키로 닫기
5. 모달에서 상태 변경(접수/처리중/완료)
6. 상태 변경 시 토스트 알림 표시
7. 토스트에서 `Undo`로 상태 변경 되돌리기
8. 상태 변경 시 카드 강조 애니메이션
9. 주문 카드에 재주문/추천 배지 및 최근 주문 요약 표시
10. 상세 모달에 최근 주문/자주 주문한 메뉴/특이사항 추천 섹션 표시
11. 추천 클릭 시 주문서/요청사항 초안에 자동 반영
12. 대시보드 레이아웃 스켈레톤(사이드바/헤더/모바일 드로어)
13. 대시보드 하위 페이지 스켈레톤(홈/주문/고객/결제/통계/설정)
14. 사이드바 접힘/펼침 상태 저장
15. 모바일 하단 네비게이션 추가
16. 주문 상세 페이지 스켈레톤 분리
17. 상세 모달에서 주문 상세 페이지 링크 제공
18. 주문 검색/필터 UI 추가 (검색어/상태/채널)
19. 주문 상세 페이지 UI 확장 (요약/고객/배송/메뉴/히스토리/추천)
20. i18n 텍스트 리소스 적용 (ko/en)

## 화면 구성
- 상단 KPI 카드 (오늘 접수/처리 대기/완료)
- 채널 비율 스트립 (Instagram/Messenger/Other)
- 필터 바 (검색, 상태, 채널)
- 칸반 3열 (접수/처리중/완료)
- 주문 카드: 고객명, 주문 내역, 주소, 요청사항, 메신저 ID, 합계
- 상세 모달: 스크린샷 프리뷰(더미), 상세 정보, 상태 변경 버튼
- 토스트: 상태 변경 피드백 + Undo
- 주문 상세 페이지: 요약/고객/배송/메뉴/히스토리/추천 카드

## 동작 요약
- 주문 카드 클릭 → 상세 모달 오픈
- 카드 내 `상세 보기` 버튼 → 상세 모달 오픈
- 모달 외부 클릭 또는 `Esc` → 모달 닫힘
- 상태 변경 버튼 클릭 → 주문 상태 이동 + 토스트 표시
- 토스트 `Undo` → 이전 상태로 복구 + 강조 애니메이션
- 검색/필터 변경 → 칸반 컬럼/카운트 즉시 반영

```js
  request: '문 앞에 두고 문자 주세요',
  orderDetail: '김치찌개 x1, 공기밥 x1',
  total: '13,000원',
  time: '14:32',
  thumbTone: 'tone-1'
}
```
